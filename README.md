# Smetomatic
Приложение для работы со строительными сметами. Пет-проект (улучшенная копия) приложения с работы.


## ТЗ

Или хотелки. Зафиксирую тут все моменты, которые хочу видеть в приложении.
- Лендинг с информацией о проекте, об изменениях
- Регистрация пользователей.
- Аутенфикация. Реализовать механизм работы с 2мя JWT токенами, аксесс и рефреш.
-- Организация работы со сметами.
- Проект - какое-то начинание. Например, строительство жилого дома.
- Объект. Это объект проекта. Внутри проекта может быть несколько объектов, дом непосредствнно, стоянка, детская площадка.
- Смета - Непосредственный документ. Объект можно также разбить на несколько смет: фундамент, крыша, интерьер.
-- Фичи
- Проект, объект и сметы имеют необязательные связи. Вполне можно вести смету без привязки к объекту.
- Проект может содержать несколько объектов. Объекты в свою очередь несколько смет.
- Смета - комплексная сущность. Есть метаинформация о смете, раздел сметы, позиция раздела сметы. Все это разные таблицы.
- Одно пространство для работы без вкладок.
- Меню слева - навигация по проектам, объектам или сметам. Позиции в меню - просто ссылки на ту или иную сущность. Можно организовать иерархию, в меню слева она будет отображаться.
- 2 режима работы с сущностями: быстрые действия и режим редактирования. Быстрые действия моментально приводят к результату, 
режим редактирования предполагает возможность в рамках сессии вернутся на шаг назад или вообще откатить изменения на начала редактирования.
- Соответствие распространенным практикам и стандартам. Возможность экспорта в отчетные документы РФ и других гос-структур.
- Сущности принадлежат конкретным пользователям. Однако, пользователи могу выдать права на взаимодействие со своими сущностями 
другим пользователям. Есть Режимы: чтения, расширенное чтение (доступны некоторые быстрые действия), редактирование.
